### [22564] Separate VDS Logging Locations

**State:** Done | **Priority:** 2 | **Tags:** Tech

**Description:**

In production, the main VDS application and the VDS Integrations API are configured to write to the same log file.  This is not ideal.  Let's separate them so we can find info more easily in the event of a production incident. 

 
While we're at it, we have a request from the Integrations team to capture the actual request bodies from the VDS Integration API requests.  They want this to help them determine how the endpoints are actually being used. 

 
We also need a way to turn the request capturing on and off without requiring a deployment.  Either a flag of some kind, or perhaps the request bodies are controlled by log verbosity. 

 
Aside from the addition of logging the request body: 
This story is not asking for a change to what is logged in the two main applications, only where it is logged. 

 
The primary intention here is to separate the logs for the applications, and within an application to separate the logs as they are currently being separated.  There was a lot of copypasta when putting the log configurations together, and you'll have to unwind it to make sure we aren't overwriting things, including archives.
