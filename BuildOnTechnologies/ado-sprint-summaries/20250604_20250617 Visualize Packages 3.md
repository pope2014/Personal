### [23822] Implement time on site analytics

**State:** Done | **Assignee:** Daniel Arwe | **Priority:** 2 | **Tags:** analytics; scripts

**Description:**

The time on site analytics spike went quite well, and the business is interested in our making this a formal solution.  

 
Refer to this spike story for the original implementation:  
https://dev.azure.com/BuildOnTechnologies/VeoDesignStudio/_workitems/edit/23559

**Acceptance Criteria:**

Builder Feature Flag:  
Create a feature flag called Time On-Site Tracking  
Description: When ON, an event is recorded at a certain frequency for all users associated for this builder. 
  

 
Heartbeat Events:  
Given that the feature flag above is ON for a builder, record a heartbeat type event that records to the event_log every [heartbeat duration] seconds  
The heartbeat duration defaults to 30s but exists in configuration so that if needed we can alter it without redeploying.  
The heartbeat event fires initially after sign-in  
Category: user_activity  
Action: heartbeat_event  
Data: context and timestamp where context is the current route info for the user and timestamp is a converted timestamp from the user's initial sign-in. 
  

 
Module Change Events:  
Given that the feature flag above is ON for a builder, when the user changes locations in the application, the same payload as the heartbeat event is recorded  
Category: user_activity  
Action: site_location_change
