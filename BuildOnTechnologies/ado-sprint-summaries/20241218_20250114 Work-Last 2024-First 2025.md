### [20358] VCMS Options Are All Set to Is Package incorrectly

**State:** Done | **Assignee:** Daniel Arwe | **Priority:** 2 | **Tags:** added; VCMS

**Repro Steps:**

In the VcmsCatalogSourceProvider class, in the GetCatlaogItems method, the results from calling the VCMS integration api are mapped to a collection of BuildCatalogItem classes. During this mapping, the code is incorrectly checking the "Type" property of a Product (set by VCMS) for the text "standard." If the type is not "standard," then VDS is assuming the product is a package and is setting the IsPackage property to true. This logic is incorrect. The VCMS no longer uses the "standard" product type. It now uses "Stock" to represent a regular product (this was per a request from the Customer Experience team). The VCMS has always supported the "Package" type though, which is what this logic should be checking for, rather than a negative check. The code should be changed to: IsPackage = o.Type.Equals("Package", StringComparison.OrdinalIgnoreCase); 

 
This is relatively easy to verify and test.
