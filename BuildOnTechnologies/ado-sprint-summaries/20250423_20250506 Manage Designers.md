### [23129] Theme Editor: Undo button fails in scenario

**State:** Done | **Priority:** 3 | **Tags:** added; fix by 6/18/25

**Repro Steps:**

1) Create a new theme. 
2) Navigate to the variables tab. 
3) Add text to the value field for any variable. 
4) Hit the undo button. 

 
Expected Result: The text written into the field is deleted, effectively reverting the field to an empty text box. 
Actual Result: Nothing happens. 

 
Commentary: 
This bug was discovered while testing the Create/Edit theme story. However, it can be reproduced as early on as the theme versioning story. The undo button works when there is an underlying base theme value to fall back to. For a long time, there always was, which is why the bug never revealed itself. However, during the versioning story, many themes, especially new ones, are now intentionally opting for no base theme. This highlights the bug.
