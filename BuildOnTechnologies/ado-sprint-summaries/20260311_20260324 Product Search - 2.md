### [29516] Product Search: Feature Tracking

**State:** Done | **Assignee:** Samuel Klepper | **Priority:** 2 | **Tags:** product search

**Description:**

Feature Description:  
As a designer or sales counselor...  
As category structures change over time, designers (or sales people) can be unaware of the location of certain items. An important customer has asked  
us if we can implement a global product search to help them locate options easier.  
 
As a buyer...  
I may be interested to determine if a specific style of product exists in my builder program. I'd like an easy way to search and determine if that is the case. It would be good if the system would locate something once I identify it exists.  
 

 
As a Product Owner/Manager:  
I want to know when this feature is being used, and how often so that I can make better decisions about continuing to invest in it.  

 
This work item implements event tracking associated with the feature so that the PO (and the full team) can get an idea of how often it is being used.

**Acceptance Criteria:**

Given that the product search feature flag is on,  
and a user performs the search  

 
log an event in the event log system with the following characteristics:  
Category: product_search   
Action: perform_search   
Data to include: org name, community, series, plan, module (OP, DMH, DS), the search term, the count of results, and if possible the search execution time (from enter to results populated)  
Notes: this allows us to know when the feature is being used, and how often a search yields results. 
 log another event when the user locates the option in the catalog context (option pricing, dmh, design selections etc)  
Category: product_search   
Action: locate_option  
Data to include: builder, option type (estimated/non estimated), option id (gpc or part id), option name, category/subcategory  
  
 
Useful metrics ideas for reporting:  
user searches per time period  
empty searches per time period - how many searches were performed that have 0 results? 
average search execution...
